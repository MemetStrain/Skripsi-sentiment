Actors

ActorTypeAccess LevelDescription
Visitor Human Public pages only (/login/, /register/) Unauthenticated person. All protected pages redirect to /login/. Identified by absence of a valid signed session cookie.
Authenticated User Human All protected pages: /, /news/, /about/, /api/prediction/ Registered user with an active session cookie. Identity (uid, username, email) stored in Firestore users collection. Session held in a signed Django cookie.
Scheduler System Full Firestore read/write via service account Automated Cloud Run job triggered by GCP Cloud Scheduler once per day. Entry point: scheduler/main.py. Has no UI. Supports --mode initial (bulk load) and --mode daily (incremental).

Authentication Use Cases

IDUse CaseActorTriggerMain FlowAlternativeImplementation
UC1 Register Account Visitor User navigates to /register/ and submits form Validate input → hash password (PBKDF2-SHA256) → write to Firestore users → redirect to /login/ Duplicate email or password <8 chars → re-render form with error views.py → register_view()
UC2 Login Visitor User submits email and password on /login/ Query Firestore for email → verify password hash → write signed session cookie → update last_login → redirect to / Wrong credentials → error message, no session created views.py → login_view()
UC3 Logout Authenticated User User clicks "Logout" in navigation bar POST /logout/ → flush session cookie → redirect to /login/ — views.py → logout_view()

Dashboard Use Cases

IDUse CaseActorTriggerMain FlowImplementation
UC4 View Price Chart Auth User Navigate to / Fetch 90-day prices + HMM states from Firestore → serialize to JSON → render Chart.js line chart in browser views.py → dashboard(), dashboard.html
UC5 View Market State Regions Auth User (via UC4) Rendered automatically with dashboard HMM states grouped into bands → Chart.js annotation plugin draws colored background: green = Bullish, red = Bearish, gray = Neutral dashboard.html JS section
UC6 View Price Statistics Auth User (via UC4) Rendered automatically with dashboard Compute current / max / min / avg close from 90-day window → display in metric cards views.py → dashboard() context
UC7 Get Price Prediction Auth User User selects model/variant/horizon, clicks "Get Prediction" JS calls GET /api/prediction/?model=…&variant=…&horizon=… → Django queries Firestore predictions → returns JSON → UI updates views.py → prediction_api()
UC8 View Model Performance Metrics Auth User (via UC7) Returned with prediction result MAPE, R², Directional Accuracy, RMSE extracted from predictions document → displayed in metrics panel views.py → prediction_api()

News & Sentiment Use Cases

IDUse CaseActorTriggerMain FlowImplementation
UC9 View News Articles Auth User Navigate to /news/ Query news_articles ordered by date desc → paginate at 9 per page → render cards with title, date, category, sentiment badge, snippet views.py → news()
UC10 Filter News by Sentiment Auth User Click filter button (All / Positive / Negative / Neutral) Pass ?sentiment=Positive → Firestore where("sentiment_label", "==", filter) → re-render filtered grid with active badge highlighted views.py → news()
UC11 Paginate News Auth User Click Next / Prev / page number Read ?page=N → Firestore offset + limit → render correct page slice with navigation controls views.py → news()

Data Pipeline Use Cases (Scheduler)

IDUse CaseModeMain FlowImplementation
UC12 Fetch Daily CPO Prices Initial: CSV parse · Daily: Investing.com API Load OHLCV rows → write only new date documents to daily_prices scheduler/price_fetcher.py
UC13 Scrape MPOB News Articles Initial: CSV load · Daily: BeautifulSoup web scrape Fetch HTML → parse articles → deduplicate by MD5(url) → write to news_articles scheduler/news_extractor.py
UC14 Run FinBERT Sentiment Analysis Daily (on new articles) Find unseen articles → GPU-batched FinBERT inference → update sentiment labels → recompute sentiment_aggregates scheduler/sentiment_runner.py
UC15 Update HMM Market States Daily (after UC12) Load full price history → compute log-returns → fit GaussianHMM (BIC) → label states → write all hmm_states scheduler/hmm_updater.py
UC16 Compute All 56 Predictions Daily (after UC12, UC14, UC15) Build feature dataset → for each of 56 combinations (4 models × 3 variants × 7 horizons): train/load → optimize → predict → evaluate → write scheduler/prediction_updater.py
UC17 Write Results to Firestore Implicit (part of all steps) Chunk into 500-doc batches → commit with batch.set() → idempotent overwrite via deterministic document IDs scheduler/firestore_writer.py
