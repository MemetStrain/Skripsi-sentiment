Entity Descriptions

USER

Firestore collection: users · Document ID: auto-generated UID

FieldTypeDescription
uidstringPrimary key — auto-generated Firestore document ID
usernamestringDisplay name chosen at registration
emailstringLowercased email address; enforced unique at application level
password_hashstringDjango make_password() hash (PBKDF2-SHA256)
created_atISO timestampAccount creation time
last_loginISO timestampUpdated on each successful login
is_activebooleanAccount status flag; inactive users cannot log in

DAILY_PRICE

Firestore collection: daily_prices · Document ID: YYYY-MM-DD

FieldTypeDescription
datestringPrimary key — the trading day (YYYY-MM-DD)
openfloatOpening price (MYR/tonne)
highfloatIntraday high price
lowfloatIntraday low price
closefloatClosing price — primary target variable for prediction
volumefloatTraded volume
updated_atISO timestampLast write time

HMM_STATE

Firestore collection: hmm_states · Document ID: {frequency}_{YYYY-MM-DD}

FieldTypeDescription
idstringComposite PK: {frequency}_{date}
datestringForeign key to DAILY_PRICE.date
frequencystringTemporal resolution — currently "Daily"
stateintDiscrete market regime: 0 = Bearish, 1 = Bullish, 2 = Neutral
state_labelstringHuman-readable label: "Bearish", "Bullish", "Neutral"
updated_atISO timestampLast HMM re-fit time

NEWS_ARTICLE

Firestore collection: news_articles · Document ID: md5(url)

FieldTypeDescription
idstringMD5 hash of article URL — deduplication key
datestringPublication date — FK to DAILY_PRICE.date
titlestringArticle headline
categorystringMPOB category (e.g. Market, Production, Export)
contentstringFull article body text
snippetstringFirst ~200 characters for news-card preview
urlstringCanonical article URL from MPOB website
sentiment_labelstringFinBERT output: "Positive", "Negative", "Neutral"
sentiment_scorefloatFinBERT confidence-weighted sentiment score [−1, +1]
scraped_atISO timestampWhen the article was scraped

SENTIMENT_AGGREGATE

Firestore collection: sentiment_aggregates · Document ID: {frequency}_{YYYY-MM-DD}

FieldTypeDescription
idstringComposite PK: {frequency}_{date}
datestringFK to DAILY_PRICE.date
frequencystringTemporal resolution — "Daily"
article_countintNumber of news articles for this date
positive_probfloatFraction of articles labeled Positive
negative_probfloatFraction of articles labeled Negative
neutral_probfloatFraction of articles labeled Neutral
sentiment_scorefloatWeighted aggregate sentiment score
combined_confidencefloatMean FinBERT confidence across all articles

PREDICTION

Firestore collection: predictions · Document ID: {model}_{variant}_{frequency}_h{horizon}

FieldTypeDescription
idstringComposite PK encoding model config and horizon
last_actual_datestringFK — last known price date used as input
modelstringML algorithm: "xgboost", "random_forest", "arimax", "sarimax"
variantstringOptimization method: "base", "csa", "bayesian"
frequencystringData frequency: "Daily"
horizonintForecast steps ahead: 1 to 7
last_actual_pricefloatMost recent known CPO closing price (MYR/tonne)
predicted_datestringDate of the forecasted price
predicted_pricefloatForecasted CPO closing price (MYR/tonne)
mapefloatMean Absolute Percentage Error (%) on test set
r2floatR² coefficient of determination on test set
rmsefloatRoot Mean Squared Error on test set
directional_accuracyfloat% of correct up/down movement predictions
computed_atISO timestampWhen the prediction was generated by the scheduler

Relationship Descriptions

RelationshipCardinalityDescription
DAILY_PRICE → HMM_STATEOne-to-ManyOne price date maps to one HMM state per frequency. Currently only Daily frequency gives a 1:1 effective mapping.
DAILY_PRICE → SENTIMENT_AGGREGATEOne-to-ManyOne price date has one sentiment aggregate per frequency, aggregated from all news articles on that date.
NEWS_ARTICLE → SENTIMENT_AGGREGATEMany-to-OneMultiple news articles published on the same date are folded into a single aggregate feature row.
DAILY_PRICE → PREDICTIONOne-to-ManyOne price date can be last_actual_date for up to 84 different model configuration predictions.
USER (self)N/AUsers are independent; no ownership or foreign key links to price/prediction data. All data is shared once logged in.

Note: Firestore is a document-oriented NoSQL database. There are no enforced foreign keys — relationships are maintained by matching date strings across collections. Document IDs are designed as idempotent keys: re-running the scheduler overwrites rather than duplicates documents.
