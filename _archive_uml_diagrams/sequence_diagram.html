1. User Registration

The full message exchange between the browser, Django view, and Firestore when a visitor creates a new account. Covers both the happy path and the two error alternatives (duplicate email, weak password). 

sequenceDiagram
    actor U as User (Browser)
    participant DV as Django View
    participant FS as Firestore users

    U->>DV: GET /register/
    DV-->>U: Render register.html (empty form)

    U->>DV: POST /register/ username, email, password

    DV->>DV: Validate password length >= 8

    alt Password too short
        DV-->>U: Re-render form with error Password must be at least 8 characters
    end

    DV->>FS: Query where email == submitted_email

    alt Email already registered
        FS-->>DV: Returns existing document
        DV-->>U: Re-render form with error Email already registered
    else Email available
        FS-->>DV: Returns empty result

        DV->>DV: make_password hashes to PBKDF2-SHA256

        DV->>FS: collection users add uid username email password_hash created_at is_active true

        FS-->>DV: Document reference auto UID

        DV-->>U: HTTP 302 redirect to /login/
    end

2. User Login and Dashboard Load

A combined flow covering login (with its Firestore credential check and session cookie creation) followed immediately by the dashboard page load (two Firestore queries, stats computation, template render, and client-side Chart.js initialisation). 

sequenceDiagram
    actor U as User (Browser)
    participant MW as Auth Middleware
    participant DV as Django View
    participant FS as Firestore
    participant TM as Template Engine

    U->>DV: POST /login/ email, password

    DV->>FS: Query users where email == submitted_email

    alt User not found
        FS-->>DV: Empty result
        DV-->>U: Re-render login.html with error Invalid credentials
    else User found
        FS-->>DV: User document uid username password_hash is_active

        DV->>DV: check_password(submitted, hash)

        alt Password mismatch
            DV-->>U: Re-render login.html with error
        else Password correct
            DV->>DV: Write _uid _username _email to signed session cookie

            DV->>FS: Update users uid last_login = now()

            DV-->>U: HTTP 302 redirect to /dashboard/
        end
    end

    U->>MW: GET /dashboard/ with session cookie

    MW->>MW: Decode signed cookie extract _uid _username _email

    MW->>MW: Check in-memory UID cache

    alt UID not in cache
        MW->>FS: users uid .get()
        FS-->>MW: User document
        MW->>MW: Add UID to in-memory cache
    end

    MW->>MW: Build FirestoreUser set request.user

    MW->>DV: Pass request to dashboard()

    DV->>FS: daily_prices order_by date desc limit 90
    FS-->>DV: 90 price documents

    DV->>FS: hmm_states where frequency==Daily and date in date_list
    FS-->>DV: HMM state documents

    DV->>DV: Compute current max min avg price

    DV->>DV: Serialize prices and HMM states to JSON arrays

    DV->>TM: render dashboard.html with context
    TM-->>U: Full HTML page with inline JSON

    U->>U: Chart.js parses inline JSON draws price line chart

    U->>U: Annotation plugin draws HMM state colored bands

3. Prediction API Call

The interaction when a logged-in user requests a prediction from the dashboard. The web server never runs ML inference — it performs a single Firestore document read and returns the pre-computed result as JSON. 

sequenceDiagram
    actor U as User (Browser)
    participant JS as Dashboard JavaScript
    participant MW as Auth Middleware
    participant DV as Django View prediction_api
    participant FS as Firestore predictions

    U->>JS: Selects Model Variant Horizon and clicks Get Prediction

    JS->>JS: Validate all dropdowns selected

    alt Missing selection
        JS-->>U: Show inline warning Please select all options
    else All selected
        JS->>JS: Show loading spinner

        JS->>DV: GET /api/prediction/ model=xgboost variant=csa frequency=Daily horizon=3

        MW->>MW: Verify session cookie set request.user

        alt Unauthenticated
            DV-->>JS: HTTP 302 redirect to /login/
        else Authenticated
            DV->>DV: Extract query params model variant frequency horizon

            DV->>DV: Build document ID xgboost_csa_Daily_h3

            DV->>FS: predictions.document xgboost_csa_Daily_h3 .get()

            alt Document not found
                FS-->>DV: DocumentSnapshot exists=False
                DV-->>JS: JSON success false error No prediction available
                JS-->>U: Show error card
            else Document found
                FS-->>DV: DocumentSnapshot with all fields

                DV->>DV: Extract predicted_price predicted_date mape r2 directional_accuracy computed_at

                DV-->>JS: JSON success true model xgboost variant csa horizon 3 predicted_price 3842.50 predicted_date 2025-04-18 mape 2.14 r2 0.91 directional_accuracy 68.5 computed_at timestamp

                JS->>JS: Hide loading spinner

                JS->>U: Update DOM prediction result card model info card metrics display
            end
        end
    end

4. Daily Scheduler Pipeline Run

The complete message sequence for the automated daily pipeline: from Cloud Scheduler trigger through six steps (price fetch, news scrape, sentiment analysis, HMM update, 56-prediction ML loop, and Firestore writes) to completion. 

sequenceDiagram
    participant GCS as Cloud Scheduler
    participant CR as Cloud Run scheduler/main.py
    participant API as External APIs
    participant ML as ML Models
    participant FS as Firestore

    GCS->>CR: HTTP POST trigger daily cron

    CR->>CR: Parse --mode daily and initialise Firestore client

    Note over CR,FS: Step 1 Prices

    CR->>FS: daily_prices get latest date
    FS-->>CR: Last stored date

    CR->>API: Fetch CPO prices since last date Investing.com API
    API-->>CR: New OHLCV rows

    CR->>FS: Batch write new daily_prices documents doc ID = date
    FS-->>CR: Batch commit ACK

    Note over CR,FS: Step 2 News

    CR->>API: Scrape MPOB news website BeautifulSoup requests
    API-->>CR: HTML article list

    CR->>CR: Parse articles and deduplicate by MD5 url

    CR->>FS: Batch write new news_articles documents doc ID = MD5
    FS-->>CR: Batch commit ACK

    Note over CR,ML: Step 3 Sentiment

    CR->>FS: Query news_articles where sentiment_label == null
    FS-->>CR: Unlabelled articles

    CR->>ML: Run FinBERT inference batched GPU
    ML-->>CR: sentiment_label and sentiment_score per article

    CR->>FS: Update news_articles with labels
    CR->>FS: Batch write sentiment_aggregates grouped by date
    FS-->>CR: Batch commit ACK

    Note over CR,ML: Step 4 HMM States

    CR->>FS: Load all daily_prices full history
    FS-->>CR: All price documents

    CR->>ML: Compute log-returns and fit GaussianHMM BIC selection and label states
    ML-->>CR: date to state mapping

    CR->>FS: Batch write all hmm_states doc ID = Daily_YYYY-MM-DD
    FS-->>CR: Batch commit ACK

    Note over CR,ML: Step 5 Predictions 56 combinations

    CR->>FS: Load prices and HMM and sentiment for feature engineering
    FS-->>CR: All required documents

    CR->>CR: Build prediction feature dataset create_prediction_dataset.py

    loop For each of 56 model combinations
        CR->>ML: Train or load model XGBoost RF ARIMAX SARIMAX
        CR->>ML: Run hyperparameter optimisation CSA or Bayesian if not base
        ML-->>CR: Optimised model

        CR->>ML: Generate h-step ahead prediction
        ML-->>CR: predicted_price and predicted_date

        CR->>ML: Evaluate on test split
        ML-->>CR: mape r2 rmse directional_accuracy

        CR->>FS: Write predictions document doc ID = model_variant_freq_hN
        FS-->>CR: Write ACK
    end

    CR-->>GCS: Pipeline complete HTTP 200 response
