👤 User Swimlane

  • user action: Input parameter (Days, Model)
  • click 'Run Prediction': Submit request ke backend
  • POST /api/predict: HTTP request dengan body JSON
  • view result: Lihat Chart & Metrics dari response
  • Error handling: Tampil error jika ada masalah

⚙️ Backend Swimlane

  • API Ready: Server listening untuk request
  • Authentication: Check user session/token (JWT)
  • Validation: Validate input parameters (Days > 0, Model valid)
  • Query Data: Request historical data dari Firestore
  • ML Pipeline: Preprocessing → Load Model → Inference → Metrics
  • Response: JSON dengan prediction array & metrics (MAPE, R²)

💾 Database Swimlane

  • Firestore Query: Query collections (price_marketstate, news_sentiment)
  • Operations: .get(), .where(), .orderBy()
  • Return: Documents array dengan fields (date, close, open, high, low, state)
  • Dataset: Dikirim kembali ke backend untuk preprocessing

⏰ Scheduler Thread

  • Cron Job: Berjalan otomatis setiap hari (00:00 UTC)
  • Trigger: Memulai scraping task
  • Background: Tidak memerlukan user interaction

🤖 Scraper Thread

  • Fetch: Ambil halaman MPOB website
  • Parse: Extract artikel berita dari HTML
  • Check: Verifikasi artikel baru
  • FinBERT: Analisis sentimen dengan model pre-trained
  • Save: Simpan hasil ke database

  • Add Document: collection('news_sentiment').add()
  • Update Field: serverTimestamp() untuk lastScrapedAt

👨‍💼 Admin Thread

  • Access: Admin dashboard (role-based)
  • Upload: Select & upload CSV file
  • Wait: Processing di backend
  • Result: View success atau error message

⚙️ Backend Thread

  • Receive: Terima file upload
  • Validate: Check format & data validity
  • Parse: Parse Indonesian CSV headers
  • Map: Convert ke format English
  • Trigger: HMM recalculation

💾 Firebase Thread

  • Batch Operation: db.batch() untuk atomic writes
  • Update: batch.update() untuk price documents
  • HMM: Kalkulasi di backend (bukan di Firebase)
  • Set States: batch.set() untuk market states
  • Commit: batch.commit() - all or nothing

🎨 UML Activity Diagram Notation

Swimlane: Vertical lanes untuk setiap actor (User, Backend, Database)

Solid Arrows (→): Control flow dalam swimlane yang sama

Dashed Arrows (- - →): Communication antar swimlane (cross-thread)

Decision Node (◇): Diamond untuk conditional branching (Yes/No)

Start Node (●): Filled black circle - initial state

End Node (◉): Bullseye (double circle) - final state

API Ready: Initial state dengan red border (synchronization point)
