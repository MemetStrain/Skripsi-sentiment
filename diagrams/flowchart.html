Pendahuluan

Flowchart memperlihatkan alur kontrol per-langkah dengan titik keputusan (decision). Dua alur utama didokumentasikan: (1) pipeline scheduler harian yang dijalankan oleh Operator lewat CLI, dan (2) alur request dashboard saat Pengunjung membuka halaman utama / memuat trail forecast.

1 — Pipeline Scheduler Harian (--mode daily)

Lima langkah berurutan dengan keputusan skip-if-current di setiap langkah. Sumber utama (single source of truth) adalah tiga CSV lokal di cpo/ dan news/; Firestore hanya dijadikan cermin downstream.

2 — Request Dashboard Pengunjung

Dua sub-alur terpisah: render HTML awal (server-side, sinkron) dan pemuatan trail forecast (asinkron, lewat /api/forecasts/). Inferensi XGBoost berjalan live di proses Django; model di-cache lewat @lru_cache sehingga pkl hanya dimuat sekali per proses gunicorn / Vercel.
