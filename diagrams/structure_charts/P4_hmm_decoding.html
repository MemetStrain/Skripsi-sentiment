Structure Chart — P4 HMM State Decoding

Dekomposisi modul proses 4.0 (DFD Level 1) — decoding state HMM dengan parameter beku + forward filter.

← DFD Structure Charts (Index) Flowcharts (Index) P1 Price P2 News P3 Sentiment P4 HMM P5 Dashboard

Pendahuluan

Proses P4 — HMM State Decoding dari DFD Level 1 diimplementasikan oleh fungsi update_hmm_states(db) di scheduler/hmm_updater.py. P4 tidak pernah me-refit HMM — parameter beku dimuat dari Firestore D5 (hmm_models/Daily), fitur dibangun ulang dari D1 (daily_prices), kemudian online forward filter menetapkan state per tanggal yang ditulis ke D4 (hmm_states). Akar struktur chart ini adalah update_hmm_states.

Catatan Implementasi

  • Frozen params, no refit: parameter HMM (D5) dipublikasi ulang hanya saat --mode rebuild-hmm; pemanggilan harian tidak pernah memanggil .fit() pada GaussianHMM.
  • Forward filter, bukan Viterbi: state pada waktu t ditentukan dari argmax_i P(q_t = i | O_1..O_t) sehingga state historis tidak berubah seiring waktu (Viterbi melakukan smoothing yang membuat state masa lalu bergerak setiap hari).
  • Lima fitur dibangun ulang dari close: Log_Return_Z, Volatility_Z, RSI_norm, MACD_Z, BB_Width_Z. _rolling_zscore menggunakan window 252 hari (default norm_window).
  • Idempotensi via filter D4: existing_dates di-stream dulu agar hanya tanggal baru yang ditulis ke hmm_states kecuali write_existing=True.
