Pendahuluan

Proses P5 — Dashboard Web dimulai dari python manage.py runserver yang menyalakan Django development server di 127.0.0.1:8000. Setiap HTTP request akan di-dispatch oleh URL router (config/urls.py → web/urls.py) ke salah satu dari empat view: dashboard, news, about, atau forecasts_api. Keempat view bersifat read-only: tiga view pertama melakukan read-and-render template, sedangkan forecasts_api hanya membaca satu dokumen Firestore forecasts/latest dan mengembalikannya sebagai JSON.

Inferensi XGBoost tidak lagi dijalankan saat request — terlalu berat untuk fungsi serverless Vercel. Inferensi dipre-komputasi offline oleh skrip website/precompute_forecasts.py yang menulis hasilnya ke data store D8 (forecasts/latest). Flowchart ini menampilkan empat sub-alur view, masing-masing diakhiri respons HTTP. Sub-alur forecasts_api kini ringkas: satu pembacaan Firestore dengan dua jalur error fallback (503 bila dokumen belum ada, 500 untuk exception lain).

Catatan Implementasi

  • Tanpa autentikasi: seluruh endpoint publik dan read-only. Tidak ada login, sesi, atau CSRF check untuk GET API.
  • Forecast dipre-komputasi: forecasts_api tidak menjalankan inferensi — ia hanya membaca dokumen forecasts/latest (D8) dari Firestore dan mengembalikan field payload apa adanya. Inferensi XGBoost dijalankan offline oleh website/precompute_forecasts.py (memanggil predictor.compute_forecast_trails), karena stack ML ~930 MB terlalu berat untuk fungsi serverless Vercel.
  • Fallback 503: bila skrip precompute belum pernah dijalankan, dokumen forecasts/latest belum ada → endpoint membalas 503 dan dashboard tetap merender (chart overlay forecast kosong).
  • Composite-index avoidance: news view sengaja meng-stream seluruh dokumen news_articles tanpa filter Firestore — filter dan sort dilakukan di Python — untuk menghindari kewajiban composite index (date + sentiment).
  • Pagination clamp: page_number = max(1, min(page_number, total_pages)) menjamin parameter di luar rentang valid tetap merender halaman batas atas/bawah (tanpa 404).
  • Cross-reference: dekomposisi modul (caller→callee) lihat Structure Chart P5.
