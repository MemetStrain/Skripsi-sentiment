Flowchart — Dashboard Web (Proses 6.0)

Alur kontrol detail untuk proses 6.0 — URL routing, dashboard render, dan endpoint forecast read-only (tanpa inferensi ML di jalur request).

← DFD P5 Structure Chart Flowcharts (Index) Pseudocodes (Index) Orchestration P3 Sentiment P4 HMM P5 Dashboard

Pendahuluan

Proses 6.0 — Dashboard Web dimulai dari python manage.py runserver yang menyalakan Django development server di 127.0.0.1:8000. Setiap HTTP request akan di-dispatch oleh URL router (config/urls.py → web/urls.py) ke salah satu dari empat view: dashboard, news, about, atau forecasts_api. Keempat view bersifat read-only dan tidak mengimpor modul ML (joblib / numpy / pandas / xgboost).

Inferensi XGBoost dipre-komputasi oleh scheduler (Proses 5.0 — scheduler/precompute_forecasts.py memanggil prediction/inference.py::compute_forecast_trails) dan ditulis ke dua dokumen Firestore: forecast_meta/Daily (winners + metrics dalam payload_json) dan koleksi forecasts (satu dokumen skalar per anchor × horizon). Flowchart ini menampilkan empat sub-alur view: dashboard menggabungkan 3 query konteks dengan baca forecast_meta/Daily untuk badge metrik; forecasts_api meng-clamp parameter, baca forecast_meta/Daily, men-stream koleksi forecasts, filter di Python, dan merakit ulang payload trail dengan shape yang persis sama dengan jalur lama.

Catatan Implementasi

  • Tanpa autentikasi: seluruh endpoint publik dan read-only. Tidak ada login, sesi, atau CSRF check untuk GET API.
  • Forecast dipre-komputasi oleh scheduler: forecasts_api tidak menjalankan inferensi — ia membaca forecast_meta/Daily (D9) untuk winners + metrics dan men-stream koleksi forecasts (D8, ~1.4k doc skalar) lalu memfilter di Python berdasarkan predicted_date dan horizon, dan merakit ulang trails dengan shape yang sama dengan jalur inferensi lama. Inferensi XGBoost dijalankan oleh proses 5.0 (scheduler/precompute_forecasts.py memanggil inference.compute_forecast_trails) karena stack ML ~930 MB terlalu berat untuk fungsi serverless Vercel.
  • Fallback 503: bila scheduler belum pernah dijalankan, forecast_meta/Daily dan/atau koleksi forecasts kosong → endpoint membalas 503 dengan pesan yang menyebut scheduler/main.py; dashboard tetap merender (chart overlay forecast kosong, badge metrik N/A).
  • Composite-index avoidance: news view sengaja meng-stream seluruh dokumen news_articles tanpa filter Firestore — filter dan sort dilakukan di Python — untuk menghindari kewajiban composite index (date + sentiment).
  • Pagination clamp: page_number = max(1, min(page_number, total_pages)) menjamin parameter di luar rentang valid tetap merender halaman batas atas/bawah (tanpa 404).
  • Cross-reference: dekomposisi modul (caller→callee) lihat Structure Chart P5.
