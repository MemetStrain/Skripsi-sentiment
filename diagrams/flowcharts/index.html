Pendahuluan

Dokumentasi flowchart pada proyek ini mengikuti pendekatan hibrid (Option C): satu flowchart orkestrasi yang menggambarkan rangkaian pipeline scheduler (P1 → P2 → P3 → P4), ditambah tiga flowchart detail untuk proses yang memiliki logika kontrol non-trivial (cabang keputusan, loop, error path): P3 (Sentiment Analysis & Aggregation), P4 (HMM State Decoding), dan P5 (Dashboard Web). Proses P1 (Price Acquisition) dan P2 (News Acquisition) bersifat sebagian besar linear dengan hanya satu skip-if-current guard masing-masing, sehingga sudah cukup terdokumentasi di dalam flowchart orkestrasi.

Pendekatan ini sejalan dengan rekomendasi Pressman & Maxim (2020) untuk hanya membuat graphical notation di tempat kompleksitas kontrol benar-benar memerlukan visualisasi — bukan untuk setiap fungsi. Tujuannya adalah meratakan tingkat detail antara dokumentasi DFD, Structure Chart, dan Flowchart agar konsisten: tiga proses teratas (P3–P5) mendapat detail per-proses; dua proses bawah (P1–P2) cukup dengan tampilan orkestrasi.

Konvensi Notasi (berlaku untuk semua flowchart)

  • Notasi mengikuti ANSI/ISO standar dalam Mermaid: kapsul = start/end, kotak = process, belah ketupat = decision.
  • Warna konsisten: startend krem (terminator), process teal (langkah aksi), decision krem-kuning (cabang), error merah (error / fallback path).
  • Setiap flowchart detail (P3, P4, P5) memuat cross-reference ke structure chart P-yang-sama untuk dekomposisi modulnya.
  • Identifier kode di node ditulis dalam bahasa Inggris (mengikuti kode sumber); narasi dan label panah dalam bahasa Indonesia.
