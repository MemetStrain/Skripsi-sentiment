Note: DailyMarketData and NewsData are logically related via time-series analysis (no physical FK).

Database Design Notes:

  • Users (Django Auth Model): Standard Django authentication model. The is_staff field is critical for distinguishing Admin vs User access control.
  • DailyMarketData (Merged Entity): Combines raw market data (OHLCV) with machine learning results (HMM state & probability) in one table for optimal query performance. The date field has a unique constraint ensuring one record per trading day.
  • NewsData (Sentiment Analysis): Stores news articles with sentiment analysis results from FinBERT model. Each article is analyzed independently and timestamped via published_date.
  • ActivityLog (Audit Trail): Tracks user actions for security auditing and system monitoring. Examples: "Uploaded CSV", "Ran Prediction", "Updated HMM Model". This table creates the relational structure needed for academic integrity.
  • Relationships:
    • Physical Relationship: Users → ActivityLog (One-to-Many via user_id foreign key)
    • Logical Relationship: DailyMarketData ↔ NewsData (Linked by time-series for correlation analysis, no foreign key)
  • HMM States: 0 = Bearish (downward trend), 1 = Bullish (upward trend), 2 = Neutral (sideways market)
  • Sentiment Labels: Classified as 'Positive', 'Negative', or 'Neutral' based on sentiment_score threshold from FinBERT
  • Design Philosophy: Procedural Django approach - data tables are kept modular, related by time rather than rigid foreign keys, allowing flexible data ingestion and analysis.
