External Entities

EntityRoleDirectionNotes
Web Visitor End user of the public dashboard Two-way (HTTP request / HTTP response) No authentication. Anyone with the URL can browse the dashboard, news, and call the forecast API.
Maintainer Operator who refreshes data Two-way (CLI invocation / log output) Runs scheduler/main.py manually on a workstation. Not part of the deployed runtime.
MPOB Website News article source Inbound (scraped HTML) Public site at the Malaysian Palm Oil Board. Pulled by the scheduler's news scraper.
Investing.com CPO price quote source Inbound (HTTP / library call) Daily OHLCV pulled via investiny. Provides the canonical close used for forecasting.

Notes

  • Only two human actor classes interact with the system: anonymous visitors (read) and the maintainer (refresh). There is no admin role inside the web app — auth was removed on 2026-05-05.
  • Both data-source entities (MPOB, Investing.com) are accessed only via the local scheduler, never directly from the deployed web frontend.
  • The "system" boundary at this level includes the deployed Django app on Vercel, the Firestore database, the local scheduler script, and the bundled XGBoost / HMM model artefacts. Level 1 splits these apart.
