Pendahuluan

Structure chart memperlihatkan dekomposisi modul dari titik masuk utama menuju fungsi-fungsi pendukung. Notasi: kotak menyatakan modul/fungsi, anak panah menyatakan pemanggilan (caller → callee), dan label panah menyatakan data yang dilewatkan. Dua hierarki utama dipetakan: pipeline scheduler (offline / per-hari) dan website (per-request HTTP).

1 — Hierarki Pipeline Scheduler

Akar: scheduler/main.py::main dengan flag --mode daily. Mode initial memuat CSV bulk ke Firestore; mode rebuild-hmm mempublikasikan ulang parameter HMM beku.

2 — Hierarki Website (Django)

Akar: website/config/urls.py mendelegasikan ke web/urls.py. Empat endpoint publik: dashboard, news, about, api/forecasts/. Tidak ada autentikasi. Website bersifat read-only: tidak ada impor ML (joblib / numpy / pandas / xgboost) — semua inferensi dijalankan oleh scheduler (Proses 5.0) dan website hanya membaca dokumen Firestore yang sudah dipre-komputasi.
