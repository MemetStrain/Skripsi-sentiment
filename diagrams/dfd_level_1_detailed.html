⚙️ P1: Authentication

  • Input: Credentials dari User/Admin
  • Proses: Validasi login dengan Users DB
  • Output: Session Token untuk akses sistem
  • Teknologi: JWT/Session Management

⚙️ P2: News Automation Pipeline

  • Input: Raw news dari MPOB (scraping)
  • Proses: FinBERT Sentiment Analysis otomatis
  • Output: Skor sentimen (positif/negatif/netral)
  • Jadwal: Background scheduler (harian)

⚙️ P3: Price & Market State Management

  • Input: Data OHLC dari Admin
  • Proses: HMM untuk hidden states (Bullish/Bearish)
  • Output: Update Price_MarketState DB
  • Akses: Admin only (role-based)

⚙️ P4: Prediction Inference

  • Input: Request User + data dari D2 & D3
  • Proses: Pre-trained Model (Random Forest/XGBoost)
  • Output: Prediksi harga t+1, metrik, grafik
  • Real-time: Response dalam hitungan detik

💾 D1: Users DB

  • Isi: User credentials, roles (User/Admin)
  • Struktur: user_id, username, password_hash, role
  • Akses: P1 (Authentication)

💾 D2: News_Sentiment DB

  • Isi: Berita MPOB + skor sentimen FinBERT
  • Struktur: date, news_text, sentiment_score
  • Update: Otomatis via P2 (harian)
  • Digunakan: P4 untuk fitur prediksi

💾 D3: Price_MarketState DB

  • Isi: Harga historis + HMM hidden states
  • Struktur: date, open, high, low, close, state
  • Update: Manual via Admin (P3)
  • Digunakan: P4 untuk fitur prediksi

🔑 Poin Penting Aliran Data

  • Garis Solid (→): Aliran data aktif/langsung
  • Garis Putus-putus (- -): Pembacaan data historis (read-only untuk kalkulasi)
  • Dua Arah (↔): Read & Write operations (contoh: P1 dengan D1)
  • Fokus Inference: Model ML sudah pre-trained, hanya melakukan prediksi real-time
