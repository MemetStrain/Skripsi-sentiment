Pendahuluan

Diagram ini memodelkan aliran data Sistem Prediksi Harga CPO mengikuti paradigma Function-Based View (FBV) — setiap proses adalah fungsi yang menerima input, menghasilkan output, dan berinteraksi dengan penyimpanan data tanpa keadaan internal (state) tersembunyi. Level 0 memperlihatkan sistem sebagai satu proses tunggal dengan entitas eksternal di sekitarnya; Level 1 mendekomposisi proses tersebut menjadi enam fungsi utama yang masing-masing dipetakan langsung ke modul Python di kode sumber.

DFD Level 0 — Diagram Konteks

Sistem dipandang sebagai satu kotak hitam (Proses 0). Empat entitas eksternal dan Pengunjung (publik) berinteraksi dengannya. Tidak ada proses internal atau data store yang ditampilkan pada level ini.

DFD Level 1 — Dekomposisi Sistem

Proses 0 didekomposisi menjadi enam fungsi utama. Lima proses pertama (1.0–5.0) dipicu oleh Operator melalui CLI scheduler (satu invocation, lima langkah berurutan); proses 6.0 dipicu oleh Pengunjung Publik melalui HTTP request. Inferensi XGBoost berjalan sebagai langkah terakhir scheduler (Proses 5.0) — modul scheduler/precompute_forecasts.py memanggil prediction/inference.py::compute_forecast_trails secara in-process dan menulis hasilnya ke dua data store: D8 forecasts (dokumen per-titik) dan D9 forecast_meta (winners + metrics). Setiap proses tertaut langsung ke modul Python yang mengimplementasinya.
