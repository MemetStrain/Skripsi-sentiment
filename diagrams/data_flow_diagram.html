Pendahuluan

Diagram ini memodelkan aliran data Sistem Prediksi Harga CPO mengikuti paradigma Function-Based View (FBV) — setiap proses adalah fungsi yang menerima input, menghasilkan output, dan berinteraksi dengan penyimpanan data tanpa keadaan internal (state) tersembunyi. Level 0 memperlihatkan sistem sebagai satu proses tunggal dengan entitas eksternal di sekitarnya; Level 1 mendekomposisi proses tersebut menjadi lima fungsi utama yang masing-masing dipetakan langsung ke modul Python di kode sumber.

DFD Level 0 — Diagram Konteks

Sistem dipandang sebagai satu kotak hitam (Proses 0). Empat entitas eksternal dan Pengunjung (publik) berinteraksi dengannya. Tidak ada proses internal atau data store yang ditampilkan pada level ini.

DFD Level 1 — Dekomposisi Sistem

Proses 0 didekomposisi menjadi lima fungsi utama. Empat proses pertama (1.0–4.0) dipicu oleh Operator melalui CLI scheduler; proses 5.0 dipicu oleh Pengunjung Publik melalui HTTP request. Setiap proses tertaut langsung ke modul Python yang diimplementasinya.
