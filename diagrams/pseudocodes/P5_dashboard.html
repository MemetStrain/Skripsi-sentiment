Pseudocode — Dashboard Web (Proses 6.0) + Algoritma Precompute (Proses 5.0)

Notasi algoritmik untuk 4 view read-only di website/web/views.py, plus algoritma inferensi yang dijalankan oleh scheduler di prediction/inference.py + scheduler/precompute_forecasts.py.

← DFD P5 Flowchart P5 Structure Chart Pseudocodes (Index) Orchestration P3 Sentiment P4 HMM P5 Dashboard

Pendahuluan

Pseudocode ini mengikuti flowchart P5. Setiap HTTP request akan di-dispatch oleh Django URL router ke salah satu dari empat view, semuanya read-only. Tiga view (dashboard, news, about) melakukan read-and-render template; forecasts_api membaca forecast_meta/Daily + koleksi forecasts dari Firestore, memfilter di Python, dan merakit ulang payload trail. Algoritma inferensi XGBoost (compute_forecast_trails) dipre-komputasi oleh scheduler (Proses 5.0 — scheduler/precompute_forecasts.py memanggil prediction/inference.py) dan didokumentasikan pada bagian “Algoritma Scheduler — Precompute Forecast” di bawah.

VIEW dashboard
SOURCE: website/web/views.py :: dashboard

INPUT:
    request  // HttpRequest (GET /)

OUTPUT:
    HttpResponse (dashboard.html)

BEGIN
    TRY
        db = firestore.client()
    CATCH ValueError                                   // Firebase not initialized
        messages.error(request, "Firebase is not initialized.")
        RETURN render(request, "dashboard.html", _empty_dashboard_ctx())
    END TRY

    three_months_ago = (today() - timedelta(days=90)).isoformat()

    // (1) prices D1, last 90 days
    price_docs = db.collection("daily_prices")
                   .where("date", ">=", three_months_ago)
                   .order_by("date").stream()
    price_list = [ to_price_dict(doc) FOR doc IN price_docs ]

    // (2) HMM states D4, last 90 days (filter freq=Daily in Python)
    state_docs = db.collection("hmm_states")
                   .where("date", ">=", three_months_ago)
                   .order_by("date").stream()
    state_dict = build_state_dict(state_docs)              // keyed by date

    // (3) Merge price + state into chart_data
    chart_data = []
    FOR row IN price_list DO
        s = state_dict.get(row.date, { state: NULL, state_label: NULL })
        chart_data.append(merge(row, s))
    END FOR

    // (4) Stats over 90-day window
    stats = compute_window_stats(price_list)              // current/avg/max/min/total
    latest_date = (price_list[-1].date) IF price_list ELSE "N/A"

    // (5) winners + metrics dari forecast_meta/Daily — D9 (Firestore, bukan filesystem)
    metrics = { mape: 0, r2: 0, accuracy: 0, best_model: "N/A" }
    winners_payload = {}
    TRY
        meta_doc = db.collection("forecast_meta").document("Daily").get()
        IF meta_doc.exists THEN
            meta_raw        = meta_doc.to_dict() OR {}
            winners_payload = json.loads(meta_raw.get("payload_json", "{}"))
            h1_tag = winners_payload.winners_by_horizon.get("1")
            IF h1_tag IS NOT NULL THEN
                h1_metrics = winners_payload.metrics
                                 .get(h1_tag, {})
                                 .get("1", {}).get("CSA", {})
                IF h1_metrics IS NOT EMPTY THEN
                    metrics = extract_h1_metrics(h1_metrics, winners_payload)
                END IF
            END IF
        END IF
    CATCH Exception
        pass                                              // keep N/A defaults bila doc belum diproduksi
    END TRY

    // (6) sentiment_aggregates D3, last 90 days (best-effort)
    sentiment_list = []
    TRY
        sent_docs = db.collection("sentiment_aggregates")
                      .where("date", ">=", three_months_ago)
                      .order_by("date").stream()
        sentiment_list = [ to_sent_dict(d) FOR d IN sent_docs ]
    CATCH Exception
        pass
    END TRY

    RETURN render(request, "dashboard.html", {
        chart_data     : json.dumps(chart_data),
        metrics        : metrics,
        winners_data   : json.dumps(winners_payload),
        sentiment_data : json.dumps(sentiment_list),
        stats          : stats,
        latest_date    : latest_date,
        page_title     : "CPO Price Prediction Dashboard" })
END

VIEW news
SOURCE: website/web/views.py :: news

INPUT:
    request  // GET /news/?sentiment=...&page=...

OUTPUT:
    HttpResponse (news.html)

BEGIN
    db               = firestore.client()
    sentiment_filter = request.GET.get("sentiment")
    page_number      = int(request.GET.get("page", 1))
    items_per_page   = 9

    news_docs = db.collection("news_articles").stream()    // no Firestore filter

    news_list        = []
    sentiment_counts = { positive: 0, negative: 0, neutral: 0, total: 0 }
    FOR doc IN news_docs DO
        d     = doc.to_dict()
        label = d.get("sentiment_label", "Neutral")
        sentiment_counts.total += 1
        increment_label_count(sentiment_counts, label)
        IF sentiment_filter AND label != sentiment_filter THEN
            CONTINUE
        END IF
        news_list.append(to_news_row(d))
    END FOR

    news_list.sort(key = LAMBDA x: x.date, reverse = TRUE)

    total_news  = |news_list|
    total_pages = max(1, ceil(total_news / items_per_page))
    page_number = max(1, min(page_number, total_pages))    // clamp
    start       = (page_number - 1) * items_per_page
    news_page   = news_list[start : start + items_per_page]

    pagination = build_pagination(page_number, total_pages, window_size=5)

    RETURN render(request, "news.html", {
        news_page         : news_page,
        sentiment_counts  : sentiment_counts,
        current_filter    : sentiment_filter,
        pagination        : pagination,
        page_title        : "CPO News & Sentiment" })
END

VIEW forecasts_api

VIEW forecasts_api
SOURCE: website/web/views.py :: forecasts_api

INPUT:
    request  // GET /api/forecasts/?max_horizon=&window_days=

OUTPUT:
    JsonResponse {horizons, winners, configs, trails, metrics, tag_to_config, generated_at}
                  (200 ok / 503 forecast_meta-atau-forecasts-belum-ada / 500 other)

BEGIN
    // (1) Clamp query params
    max_horizon = clamp(int(request.GET.get("max_horizon", 7)), 1, 7)
    window_days = clamp(int(request.GET.get("window_days", 90)), 7, 365)

    TRY
        db = firestore.client()

        // (2) forecast_meta/Daily → winners + configs + metrics
        meta_doc = db.collection("forecast_meta").document("Daily").get()  // ← D9
        IF NOT meta_doc.exists THEN
            RETURN JsonResponse(
                { error: "forecast_meta/Daily not found. Run scheduler/main.py to populate forecasts." },
                status=503)
        END IF
        meta_raw     = meta_doc.to_dict() OR {}
        meta_payload = json.loads(meta_raw.get("payload_json", "{}"))

        // (3) Stream forecasts collection, filter di Python
        cutoff       = (today() - timedelta(days=window_days)).isoformat()
        trails_by_h  = {}
        FOR d IN db.collection("forecasts").stream() DO     // ← D8
            x = d.to_dict() OR {}
            IF (x.frequency OR "Daily") != "Daily" THEN CONTINUE
            h = int(x.horizon)
            IF h < 1 OR h > max_horizon THEN CONTINUE
            pred_date = x.predicted_date
            IF pred_date IS EMPTY OR pred_date < cutoff THEN CONTINUE
            trails_by_h.setdefault(h, []).append({
                anchor_date     : x.anchor_date,
                anchor_price    : round(float(x.anchor_price), 2),
                predicted_date  : pred_date,
                predicted_price : round(float(x.predicted_price), 2),
                log_return      : round(float(x.log_return), 6) })
        END FOR

        IF trails_by_h IS EMPTY THEN
            RETURN JsonResponse(
                { error: "forecasts collection empty. Run scheduler/main.py to populate forecasts." },
                status=503)
        END IF

        // (4) Group + sort + assemble — payload shape identik dengan jalur lama
        winners = meta_payload.get("winners_by_horizon", {})    // keys: str(h)
        configs = meta_payload.get("configs_by_horizon", {})
        trails  = []
        FOR h IN sorted(trails_by_h.keys()) DO
            points = sorted(trails_by_h[h], key = LAMBDA p: p.anchor_date)
            trails.append({
                horizon : h,
                tag     : winners.get(str(h), ""),
                config  : configs.get(str(h), ""),
                points  : points })
        END FOR

        RETURN JsonResponse({
            horizons      : meta_payload.get("horizons", range(1, max_horizon + 1)),
            winners       : winners,
            configs       : configs,
            trails        : trails,
            metrics       : meta_payload.get("metrics", {}),
            tag_to_config : meta_payload.get("tag_to_config", {}),
            generated_at  : meta_raw.get("generated_at", "") })
    CATCH Exception AS e
        RETURN JsonResponse({ error: str(e) }, status=500)
    END TRY
END

Algoritma Scheduler — Precompute Forecast (Proses 5.0)

Algoritma di bawah ini dijalankan oleh scheduler (Proses 5.0 di DFD), bukan oleh server web. run_daily_update dan run_initial_load di scheduler/main.py memanggil _step_precompute(db) sebagai langkah terakhir, yang membungkus precompute_and_write dalam try/except agar kegagalan inferensi tidak membatalkan langkah price / news / reconcile / HMM. Orkestrator memanggil inference.compute_forecast_trails, mem-flatten trails ke dokumen per-titik (skalar saja), lalu menulis koleksi forecasts (D8) + dokumen forecast_meta/Daily (D9). Stack ML (~930 MB) hanya dibutuhkan di lingkungan scheduler, bukan di fungsi serverless Vercel. 

PROCEDURE precompute_and_write
SOURCE: scheduler/precompute_forecasts.py :: precompute_and_write

INPUT:
    db                // Firestore client (disuplai oleh scheduler/main.py)

OUTPUT:
    int               // jumlah dokumen forecast point yang ditulis (0 jika trails kosong)

CONSTANTS:
    PRECOMPUTE_WINDOW_DAYS = 365
    PRECOMPUTE_MAX_HORIZON = 7
    FREQUENCY              = "Daily"

BEGIN
    inference = _import_inference()                  // sys.path += prediction/ → import inference

    payload = inference.compute_forecast_trails(
                  db,
                  max_horizon = PRECOMPUTE_MAX_HORIZON,
                  window_days = PRECOMPUTE_WINDOW_DAYS)

    trails = payload.get("trails", [])
    IF trails IS EMPTY THEN
        log_warning("no trails; nothing written")
        RETURN 0
    END IF

    // Flatten trail (nested) → list per-titik dengan field skalar
    points = []
    FOR trail IN trails DO
        FOR p IN trail.points DO
            points.append({
                frequency       : FREQUENCY,
                horizon         : int(trail.horizon),
                tag             : str(trail.tag),
                config          : str(trail.config),
                anchor_date     : str(p.anchor_date),
                anchor_price    : float(p.anchor_price),
                predicted_date  : str(p.predicted_date),
                predicted_price : float(p.predicted_price),
                log_return      : float(p.log_return) })
        END FOR
    END FOR

    n = write_forecasts(db, points)                  // → D8 (~1.4k doc, ID = Daily_h{h}_{anchor})

    write_forecast_meta(db, {                        // → D9 (forecast_meta/Daily)
        frequency          : FREQUENCY,
        generated_at       : payload.get("generated_at", ""),
        max_horizon        : PRECOMPUTE_MAX_HORIZON,
        window_days        : PRECOMPUTE_WINDOW_DAYS,
        winners_by_horizon : payload.get("winners", {}),
        configs_by_horizon : payload.get("configs", {}),
        metrics            : payload.get("metrics", {}),
        tag_to_config      : payload.get("tag_to_config", {}),
        horizons           : payload.get("horizons", []) })

    _delete_legacy_doc(db)                           // hapus forecasts/latest legacy (idempotent)
    RETURN n
END


ALGORITHM compute_forecast_trails
SOURCE: prediction/inference.py :: compute_forecast_trails

INPUT:
    db, max_horizon, window_days

OUTPUT:
    payload (JSON-serializable dict): {
        horizons, winners, configs, trails[], metrics,
        tag_to_config, generated_at }

BEGIN
    winners_payload = load_winners()                              // lru_cache(1) ← D7
    winners = { int(h): tag FOR (h, tag) IN winners_payload.winners_by_horizon.items() }
    configs = { int(h): cfg FOR (h, cfg) IN winners_payload.configs_by_horizon.items() }

    df = build_inference_frame(db)
    IF df IS EMPTY THEN
        RETURN { horizons: [], winners: {}, configs: {}, trails: [], metrics: {} }
    END IF

    dates_sorted = df.Date.tolist()
    last_idx     = |dates_sorted| - 1
    next_td      = _next_trading_day_after(dates_sorted[last_idx])

    cutoff     = df.Date.max() - timedelta(days = window_days)
    window_idx = indices_where(df.Date >= cutoff)

    trails = []
    FOR h = 1 TO max_horizon DO
        tag = winners.get(h)
        IF tag IS NULL THEN
            log_warning("no winner for h=" + h)
            CONTINUE
        END IF

        TRY
            (model, scaler, feature_cols) = load_model(tag, h, "csa")  // lru_cache(64)
        CATCH Exception
            log_warning("failed load_model h=" + h + " tag=" + tag)
            CONTINUE
        END TRY

        points = []
        FOR i IN window_idx DO
            target_idx = i + h
            IF target_idx > last_idx + 1 THEN CONTINUE
            pred_date = dates_sorted[target_idx] IF target_idx <= last_idx
                        ELSE next_td

            row = df.iloc[[i]]
            TRY
                X = select_feature_matrix(row, feature_cols)
            CATCH ValueError
                CONTINUE                                        // kolom fitur tidak lengkap
            END TRY
            IF any_nan(X) THEN CONTINUE

            IF scaler IS NOT NULL THEN X = scaler.transform(X)
            log_ret      = float(model.predict(X)[0])
            anchor_close = float(row.Close.iloc[0])
            pred_price   = anchor_close * exp(clip(log_ret, -10, 10))

            points.append({
                anchor_date     : format_date(dates_sorted[i]),
                anchor_price    : round(anchor_close, 2),
                predicted_date  : format_date(pred_date),
                predicted_price : round(pred_price, 2),
                log_return      : round(log_ret, 6) })
        END FOR

        trails.append({ horizon: h, tag: tag, config: configs.get(h, ""), points: points })
    END FOR

    RETURN {
        horizons      : range(1, max_horizon + 1),
        winners       : { str(h): t FOR (h, t) IN winners.items() },
        configs       : { str(h): c FOR (h, c) IN configs.items() },
        trails        : trails,
        metrics       : winners_payload.get("metrics", {}),
        tag_to_config : winners_payload.get("tag_to_config", {}),
        generated_at  : utcnow_isoformat() + "Z" }
END


PROCEDURE build_inference_frame
SOURCE: prediction/inference.py :: build_inference_frame

BEGIN
    price_df     = _load_prices(db)             // ← D1
    IF price_df IS EMPTY THEN RAISE RuntimeError

    sent_df      = _load_sentiment(db)          // ← D3
    hmm_state_df = _load_hmm_state(db)          // ← D4 (Date, HMM_State, HMM_State_Label)

    df = add_price_features(price_df)
    df = add_hmm_derived_features(df)
    df.Change_Pct = df.Close.pct_change() * 100

    IF hmm_state_df IS NOT EMPTY THEN
        df = df.merge(hmm_state_df, on="Date", how="left")
        df.HMM_State       = fillna(df.HMM_State, 2)         // default Neutral=2
        df.HMM_State_Label = fillna(df.HMM_State_Label, "Neutral")
        top_labels = value_counts(df.HMM_State_Label).head(5)
        FOR state IN top_labels.index DO
            col = "HMM_" + state.replace(" ", "_").replace("-", "_")
            df[col] = (df.HMM_State_Label == state).astype(int)
        END FOR
        df = df.drop(columns=["HMM_State_Label"])
    END IF

    IF sent_df IS NOT EMPTY THEN
        df = df.merge(sent_df, on="Date", how="left")
        fillna(df[Article_Count, Positive_Prob, Negative_Prob,
                  Neutral_Prob, Sentiment_Score], 0.0)
        IF "Confidence" NOT IN df.columns THEN
            df.Confidence = max(Positive_Prob, Negative_Prob, Neutral_Prob).fillna(0.0)
        END IF
    END IF

    df = engineer_all_features(df)              // cyclical + lags + interactions
    RETURN df.sort_values("Date").reset_index(drop=TRUE)
END


PROCEDURE load_model
SOURCE: prediction/inference.py :: load_model  (@lru_cache(maxsize=64))

INPUT:
    tag, horizon, variant = "csa"

OUTPUT:
    (model, scaler, feature_cols)

BEGIN
    base        = path.join(_SAVED_MODELS_DIR, tag, "Daily", "h" + horizon, "xgboost_" + variant)
    model_path  = path.join(base, "model.pkl")
    scaler_path = path.join(base, "scaler.pkl")
    meta_path   = path.join(base, "meta.json")

    IF NOT (path.exists(model_path) AND path.exists(meta_path)) THEN
        RAISE FileNotFoundError("Model artefacts missing in " + base)
    END IF

    meta         = json.load(meta_path)
    feature_cols = list(meta.get("feature_cols", []))
    IF feature_cols IS EMPTY THEN RAISE ValueError

    model  = joblib.load(model_path)                              // ← D6
    scaler = joblib.load(scaler_path) IF path.exists(scaler_path) ELSE NULL
    RETURN (model, scaler, feature_cols)
END

Catatan Implementasi

  • Tanpa autentikasi: seluruh endpoint publik dan read-only.
  • Pemisahan website / scheduler: forecasts_api membaca forecast_meta/Daily (D9) + koleksi forecasts (D8); tidak ada impor ML di website/. Algoritma compute_forecast_trails dijalankan oleh scheduler (scheduler/precompute_forecasts.py → prediction/inference.py) sebagai langkah 5 daily run — sehingga situs Vercel tidak perlu mem-bundle atau menjalankan stack ML.
  • Cache (jalur scheduler): load_winners di-cache dengan @lru_cache(maxsize=1); load_model dengan @lru_cache(maxsize=64). Maksimum kombinasi yang dibutuhkan: 4 tags × 7 horizon × 1 varian CSA = 28 entri.
  • Idempotensi: write_forecasts memakai doc ID deterministik Daily_h{h}_{anchor} sehingga full-recompute pada window 365 hari secara aman menulis ulang dokumen yang sama (tidak ada batas 1 MB Firestore — payload disebar ke ~1.4k dokumen skalar, bukan satu blob). write_forecast_meta mem-bundle nested arrays ke payload_json (mengikuti pola write_hmm_params).
  • Convergence trail: setiap horizon h-step-ahead diakhiri pada next_td yang sama → 7 trail bertemu di titik prediksi yang sama.
  • Pagination clamp di news: page_number = max(1, min(page_number, total_pages)) — parameter di luar rentang tetap merender halaman valid (tidak 404).
  • Composite-index avoidance: news men-stream seluruh dokumen dan filter di Python — menghindari composite index (date + sentiment).
  • Cross-reference: flowchart P5, structure chart P5.
