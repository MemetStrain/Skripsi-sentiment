👨‍💼 Admin Swimlane

  • Select Menu: Akses menu "Update Price" (role-based access)
  • Upload CSV: File dengan format Indonesian headers (Tanggal, Terakhir, dll)
  • View Success: Notifikasi "Data berhasil diupdate & HMM recalculated"
  • Error Handling: Jika format invalid, tampil error & retry

⚙️ Backend Swimlane (Django)

  • Receive File: Accept multipart/form-data upload
  • Parse CSV: Read headers & validate structure
  • Validation: Check required columns (Date, Open, High, Low, Close)
  • Clean Data: Remove duplicates, handle nulls, format dates
  • Batch Write: Prepare Firestore batch operation
  • HMM Calculation: Run Hidden Markov Model algorithm
  • States: Determine Bullish (1) / Bearish (0) / Neutral (2)
  • Update Docs: Add hmm_state & hmm_prob fields
  • Audit Log: Record activity untuk compliance

💾 Database Swimlane (Firestore)

  • Collection: DailyMarketData (price documents)
  • Batch Write: db.batch() untuk atomic operations
  • Document Structure: {date, open, high, low, close, hmm_state, hmm_prob}
  • Update Fields: batch.update() untuk existing documents
  • Activity Log: Collection ActivityLog untuk audit trail
  • Commit: batch.commit() - all or nothing

🔄 Process Flow Summary

1. Upload: Admin upload CSV file dengan price data baru

2. Validation: Backend validate format & clean data

3. Batch Write: Firestore batch write raw price documents

4. HMM Calculation: Backend run Hidden Markov Model untuk determine market states

5. Update States: Firestore batch update documents dengan hmm_state & hmm_prob

6. Audit Log: Insert record ke ActivityLog collection

7. Success Response: Return JSON response ke Admin UI

8. Notification: Admin view success message

🔥 Firestore Operations

Batch Write: db.batch().set(docRef, data) untuk multiple documents

Batch Update: batch.update(docRef, {hmm_state: 1, hmm_prob: 0.85})

Add Document: collection('ActivityLog').add({timestamp, action, admin})

Commit: batch.commit() untuk atomic operation (all or nothing)

Advantage: NoSQL flexibility, automatic scaling, real-time sync
