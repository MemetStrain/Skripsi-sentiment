## 2. Target Definition Change

### What changed
In each file's `engineer_features_for_horizon()`:

```python
# BEFORE
df['Target'] = df['Close'].shift(-horizon)

# AFTER
df['Target'] = np.log(df['Close'].shift(-horizon) / df['Close'])
```

**Why:** Raw close price is non-stationary; log returns are stationary and better-behaved for ML model training. The h-step cumulative log return `log(Close_{t+h} / Close_t)` captures the direction and magnitude of price movement from the current period to h steps ahead.

**No additional NaN rows introduced:** `dropna()` already removes the last `horizon` rows.

## 3. New Alignment Assertion in `prepare_train_test`

Added at the top of `prepare_train_test` in `forecast_utils.py`:

assert df['Close'].isna().sum() == 0, "Close has NaN — alignment broken"

This ensures `close_test[i]` and `y_test[i]` are always aligned.

## 4. Metrics Changes in `calculate_metrics`

### New signature
```python
calculate_metrics(y_true_lr, y_pred_lr, close_anchor)
```
(previously: `calculate_metrics(y_true, y_pred)`)

### New metrics returned
| Metric | Description |
|---|---|
| `MAPE` | Mean Absolute Percentage Error in **price space** |
| `sMAPE` | Symmetric MAPE in price space |
| `RMSE` | Root Mean Squared Error in **price space** (MYR/tonne) |
| `Directional_Accuracy` | % correct sign predictions on log return (= price direction) |
| `R2_Price` | R² computed in price space (interpretable in MYR/tonne) |
| `R2_LogReturn` | R² computed in log-return space (for model comparison robustness) |

### Price-space inversion
```python
price_actual    = close_anchor * exp(clip(y_true_lr, -10, 10))   # guarded
price_predicted = close_anchor * exp(clip(y_pred_lr, -10, 10))   # guarded
# Note: CSV Actual_Price uses raw unclipped y_true for honest reporting
```

### CSA objectives unchanged
`csa_objective_sklearn`, `csa_objective_arimax`, `csa_objective_sarimax` still score on log-return RMSE during cross-validation (consistent with training loss).

|---|---|
